> A dropdown allows the user to select an option from a list of options in an expandable overlay.

## Anatomy

1. **Label** - a short description or prompt that lets the user know what they will be selecting.
1. **Button** - a button that opens the dropdown listbox and displays the currently selected option.
	- Default text before selection should be "Select".
	- A marker should be displayed to the right of the button's text to convey whether the dropdown listbox is open or closed.
1. **Listbox** - a container for the list of options that only appears when opened by the dropdown button.
1. **Selected option** - the currently selected option.
1. **Options** - the available options. Clicking an unselected option will select that option, deselect the currently selected option, and close the listbox.

## Usage

- Use a dropdown to select one option from a collapsed set of available options.
- By default, dropdown options are mutually exclusive. The user can only select 1 option.
- When an option is selected, the dropdown closes and the selected option updates to display that item.
- On load, the dropdown should display default text of `"Select"`. The user should be able to change it back to `"Select"` if they wish.
- <kbd>Down Arrow</kbd>/<kbd>Up Arrow</kbd> moves focus to the next/previous options, respectively.
- <kbd>Home</kbd> and <kbd>End</kbd> moves focus to the first/last options, respectively.

<CalloutSuccess title="Do">
	<ul>
		<li>Do keep dropdown options very concise.</li>
		<li>If long options are unavoidable, then they can truncate when the dropdown is in a closed state, but they should wrap fully in the open state.</li>
		<li>Do ensure that the clickable area includes the entire selected option area and the icon.</li>
		<li>Consider the most logical order to list options. Options could be listed alphabetically, chronologically, by popularity, etc.</li>
	</ul>
</CalloutSuccess>

<CalloutError title="Don't">
	<ul>	
		<li>Do not start all of the options in a dropdown with the same word or phrase.</li>
		<li>If there are more than 6 options, consider using a combo box instead.</li>
		<li>If there are 2 options, consider using [radio group](radio-group) or [switch](switch) instead.</li>
		<li>If your interface needs to contain many of these selections, use dropdowns over radio buttons.</li>
		<li>Do not use a dropdown when the user can multi-select options. Use checkboxes instead.</li>
		<li>If it is not absolutely necessary for a user to choose from a predefined set of options, then consider a [text field](text-field) for users to type their own information.</li>
	</ul>
</CalloutError>

## React API

We expose two components for the dropdown API: the [Dropdown](https://github.com/wwnorton/design-system/tree/main/packages/react/src/components/Dropdown)
and the Option, which uses the [BaseOption](https://github.com/wwnorton/design-system/tree/main/packages/react/src/components/BaseOption).

```js
import { Dropdown, Option } from '@wwnds/react';
```

### Dropdown

The `Dropdown` component does not expose a `ref` or extend a full DOM interface.

```jsx live
<Dropdown label="Choose a sitcom" sort="ascending">
	<Dropdown.Option>The Office</Dropdown.Option>
	<Dropdown.Option>Community</Dropdown.Option>
	<Dropdown.Option>Friends</Dropdown.Option>
	<Dropdown.Option>Parks and Recreation</Dropdown.Option>
	<Dropdown.Option>How I Met Your Mother</Dropdown.Option>
	<Dropdown.Option>Modern Family</Dropdown.Option>
	<Dropdown.Option disabled>The Big Bang Theory</Dropdown.Option>
	<Dropdown.Option>Arrested Development</Dropdown.Option>
</Dropdown>
```

<PropsTable from={Dropdown} />

### Option

The `Option` component extends the `React.LiHTMLAttributes<HTMLLIElement>` interface, and uses the [ARIA option role](https://www.w3.org/TR/wai-aria/#option).

Note that the `Dropdown` also exposes the `Option` as a static member (`Dropdown.Option`) for your convenience.

```jsx live
<Option>Community</Option>
```

Popover

A popover allows the user to reveal more information or non-critical tasks in an overlay.

Anatomy#

  1. Container - the popover's container overlays the main page's content.
  2. Arrow - a visual indicator that points to the related element.
  3. Title (recommended) - the title of a callout summarizes its contents.
  4. Close button (optional) - a button that will always close the popover.
  5. Body - the contents of the popover.
  6. Action buttons (optional) - one or more custom actions that will result in the popover closing.

Usage#

Use a popover for progressive disclosure of information or actions that don't belong on the main page.

Don't
Open a popover on focus.
Do
Open a popover by clicking a button.

Why: Popovers must be focused when they open, which means that opening one by focusing the trigger element would cause focus to move twice in one action. This is a violation of WCAG 3.2.1: On Focus.

Don't
Use a popover to label an element.
Do
Use a tooltip instead.

Why: Even thought they look similar, popovers and tooltips serve two very different purposes. Popovers are reserved for complex related content or tasks, whereas tooltips are used to name or describe a related element.

React API#

We expose one component for the popover API, the Popover.

Popover#

The Popover component extends the React.HTMLAttributes<HTMLElement> interface and uses Popper.js for positioning.

import { Popover } from '@wwnds/react';
